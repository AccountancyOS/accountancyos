## F. Proposed guardrails

1. **TS const enums** under `src/lib/db-constants/`:
   - `JOB_STATUS` (9 values)
   - `FILING_STATUS` (6)
   - `EMAIL_QUEUE_STATUS` (4)
   - `QUESTIONNAIRE_STATUS` (4)
   - `ENGAGEMENT_STATUS` (4)
   - `CLIENT_STATUS` / `COMPANY_STATUS` (4)
   - `PORTAL_ACCESS_STATUS` (3)
   - `TB_SNAPSHOT_STATUS` (3)
   Replace string literals; ts-strict will surface every remaining bad value.
2. **`uniqueLegacyToken()` helper** wrapping `crypto.randomUUID()` for deprecated UNIQUE token columns.
3. **BEFORE INSERT trigger on `quote_acceptance_tokens`** to backfill `organization_id` from `quote_id` if NULL (defence-in-depth even though current RPC is correct).
4. **TS helper `queueEmail()`** that calls `queue_email_safe` RPC — forbid raw `email_queue` inserts via lint rule comment + grep test.
